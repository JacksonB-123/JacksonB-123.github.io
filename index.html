Project Goal¶

The goal of this project is to investigate how the Memphis Grizzlies could hypothetically build the best roster possible while staying financially flexible. We want to combine player performance data, advanced analytics, and salary information to identify players who provide the best value relative to their contracts. We also want to explore how roster construction affects payroll and whether a team can remain under important salary thresholds such as the NBA second apron, and how reaching thresholds can limit roster flexibility.

Main Questions¶

  • Which NBA players provide the best value relative to their salaries?
  • Which players could help improve the Memphis Grizzlies without harming long-term cap flexibility?
  • How do salary structures and advanced metrics interact in roster building?

Important facts to consider before analysis:¶

Salary Rules:¶

There are 5 stages of salary in the NBA.

    1. Teams under the 154.6 million dollar cap. These teams have the most flexibility and can make roster moves freely without tax penalties. As long as teams don't go below the salary floor and above the luxury tax threshold teams can make moves freely
    1. 154.6-187.9 million dollars are teams above the cap but under the luxury tax threshold. This is where the vast majority of teams operate and there are some restrictions to being over the cap. Teams can not sign players with cap space but must use exceptions. The three exceptions used are non taxpayer midlevel exception (12-13m), bi-annual exception (4-5m)
    1. 187.9-195.9 million dollars are teams above the tax threshold but under the first apron. Teams in this bracket are paying the luxury tax which is a progressive tax. The mid level exception can still be used in this bracket.
    1. 195.9-207.8 million dollars are teams above the first apron but below the second apron. This bracket suffers significant roster flexibility restrictions. Teams in this bracket have lost rights to the full mid-level exception. Teams can now only use the Taxpayer midlevel exception of 5 million. Signs and trades are now unavailable. Teams are basically limited to minimum contracts.
    1. Teams above the second apron of 207.8 million dollars suffer extreme restrictions. This bracket is almost only used by teams that are actively contending for the title. Some of the most important restrictions are teams can not use the mid level exception at all and future draft picks can be frozen or moved to the end of the round. (AthlonSports) https://athlonsports.com/nba/luxury-tax-aprons-salary-cap-how-does-it-work

def clean_name(name):
    if pd.isna(name):
        return np.nan
    name = str(name).strip().lower()
    name = re.sub(r"\b(jr|sr|ii|iii|iv)\b\.?", "", name)
    name = re.sub(r"[^\w\s-]", "", name)
    name = re.sub(r"\s+", " ", name).strip()
    return name

salary_cap = 154_600_000
luxury_tax = 187_900_000
first_apron = 195_900_000
second_apron = 207_800_000

def classify_salary_group(payroll):
    if payroll < salary_cap:
        return "1) Under Cap"
    elif payroll < luxury_tax:
        return "2) Above Cap, Below Tax"
    elif payroll < first_apron:
        return "3) Tax, Below 1st Apron"
    elif payroll < second_apron:
        return "4) 1st Apron to 2nd Apron"
    else:
        return "5) Above 2nd Apron"

team_payroll["salary_group"] = team_payroll["team_payroll"].apply(classify_salary_group)

LIMITATIONS:¶

  • The value statistics are extremely basic and only use box score stats. This puts far too much weight on contracts and box scores rather than real in game value
  • Value doesn't just show up on the box the score. There are advanced stats that display other forms of value that I will include later in the project
  • I also have ignored some of the trade exception rules, contract lengths (expiring deals), injuries, age and trade matching rules.
  • This is just a basic start need more filtering and a better formula for value
